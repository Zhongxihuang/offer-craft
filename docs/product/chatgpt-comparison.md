# 与直接问 ChatGPT 的对比

## 对比问题

同样输入一份 JD 和一份候选人简历，用户直接问 ChatGPT 通常会得到一段综合建议，但很难稳定获得可复用的决策结构。

## 直接问 ChatGPT 的常见输出

- 优点和不足混在一起；
- 会给出一些面试建议，但优先级不清楚；
- 很少明确说明哪些差距最影响招聘判断；
- 缺少 confidence、missing evidence、fallback 状态；
- 下一步动作可能泛化，例如“加强 AI 理解”“多准备案例”。

## 本项目的差异

| 维度 | 直接问 ChatGPT | 本项目 |
|---|---|---|
| 产品形态 | 一次性聊天 | 固定 workflow |
| 决策输出 | 常见为建议段落 | 明确 apply verdict |
| 差距排序 | 不稳定 | Top 3 gaps，按招聘影响排序 |
| 证据表达 | 可能泛化 | 展示 strongest evidence / missing evidence |
| 准备建议 | 常见为原则 | 3-7 天行动计划 |
| 追问方式 | 新对话容易丢上下文 | support chat 关联 workflowId |
| 失败处理 | 取决于模型回答 | 统一错误 envelope + fallback notes |

## 面试讲法

这个项目不是为了替代 ChatGPT，而是把“求职判断”这个高频任务产品化：把输入、判断、证据、差距、行动和追问组织成稳定路径。核心价值不是模型更聪明，而是用户更容易做决定、更容易知道下一步怎么做。
