# 中文 PRD：AI 求职决策与面试准备 Agent

## 1. 背景

AI 产品、平台产品、GenAI 应用岗位的 JD 往往要求模糊但门槛不低。候选人通常会把 JD 和简历直接丢给 ChatGPT，但得到的多是泛化建议，缺少明确判断：

- 这份岗位现在值不值得投？
- 哪些差距最影响筛选和面试？
- 我应该先改简历、先补项目表达，还是直接准备面试？
- 接下来 3-7 天应该具体做什么？

## 2. 目标用户

核心用户是正在投递 AI PM、产品经理、平台产品、AI 应用产品岗位的候选人，尤其是：

- 有部分 AI 项目经验，但不知道是否足够支撑目标 JD；
- 简历里有产品、数据、运营或技术背景，但不知道如何转成岗位证据；
- 需要快速判断多个岗位投递优先级；
- 需要围绕 JD 生成更具体的面试准备计划。

## 3. 核心痛点

1. JD 信息密度高，候选人很难判断真正的 must-haves。
2. 候选人容易高估或低估自己的匹配度。
3. 传统聊天式 AI 输出长，但不一定能形成清晰决策。
4. 面试准备建议经常过于泛化，不能直接指导接下来几天的行动。
5. 候选人补充新经历后，缺少持续更新分析结果的闭环。

## 4. 为什么先做“该不该投”

“该不该投”是求职链路里最早、最高频、也最影响后续投入的判断。只有先回答这个问题，后续的简历优化、面试准备和公司研究才有优先级。

本项目把输出顺序设计为：

1. Apply verdict：立即投、调整表达后投、先准备、暂时转向。
2. Top 3 gaps：解释最影响结果的差距。
3. Evidence and confidence：说明依据和不确定性。
4. 3-7 day action plan：把建议落到具体动作。
5. Support chat/refine：继续追问或补充证据，生成更新版本。

## 5. MVP 范围

保留：

- JD 文本/文件输入；
- 简历/候选人背景文本/文件输入；
- 结构化 workflow 输出；
- Top 3 gaps 和 3-7 天行动计划；
- H2 持久化和 workflow restore；
- 中英文切换；
- workflow-linked support chat；
- demo mode 无 key 可运行。

暂不做：

- 登录/账号系统；
- 自动投递；
- 付费体系；
- 多用户权限体系；
- OCR 扫描简历；
- 完整求职 CRM 或历史看板。

## 6. 成功指标

MVP 阶段建议关注：

- workflow 完成率；
- 用户是否能在 5 秒内找到 apply verdict、Top 3 gaps 和 next action；
- 缺失输入、上传失败、恢复失败等错误是否能被用户理解并恢复；
- eval case 中强简历/弱简历/不完整输入是否给出合理差异；
- support chat 是否围绕 workflowId 继续解释和补充，而不是退化成泛聊天。
