# Job Seeking Playbook (End-to-End)

This is a practical pipeline you can run weekly.

## North star
**Goal:** convert effort into interviews efficiently by aligning:
- positioning (what role you fit)
- proof (projects)
- distribution (applications + referrals)
- performance (interviews)

---

# Step 1 — Positioning (1–2 hours)

## One-sentence pitch
> I’m a [target role] who builds [type of impact] using [skills], proven by [project result].

## Choose 1–2 target roles (avoid scattering)
Examples:
- Backend engineer (APIs + DB + reliability)
- Security/appsec (secure coding + threat modeling)
- Data/growth analyst (SQL + metrics + experiments)

## Skill proof mapping
For each role, pick 3 proof buckets:
- Core engineering (tests, CI, refactoring)
- Domain skills (backend/data/security)
- Communication (docs, design notes, demo)

---

# Step 2 — Resume (ATS + human readable)

## Bullet formula
**Action + Scope + Tools + Result**
- Built X that does Y using Z, improving metric by N% / reducing latency by Nms / saving N hours/week.

## Resume checklist
- [ ] 1 page (for early-career)
- [ ] Strong summary aligned to target role
- [ ] Skills section matches job descriptions
- [ ] Each project has measurable outcomes (even if small)
- [ ] Links: GitHub + portfolio + demo (if available)
- [ ] No vague bullets (“helped”, “worked on”)

## Keyword strategy
- Copy the job post into a note
- Extract repeated nouns/verbs (e.g., “REST”, “PostgreSQL”, “CI”, “monitoring”)
- Ensure your resume naturally contains them (truthfully)

---

# Step 3 — Portfolio (recruiter-friendly)

## Portfolio items (minimum)
- 2 strong projects (primary role-aligned)
- 1 supporting project (breadth)

## Each project must have
- Quickstart (run in 5 minutes)
- Screenshots or short demo GIF
- Architecture notes
- Tests and CI badge
- “What I learned” section

---

# Step 4 — Networking & referrals (high ROI)

## Weekly target
- 10–20 thoughtful messages/week
- 2 informational chats/week (if possible)

## Outreach script (short)
- Context: why them
- Value: what you admire / a specific question
- Ask: 10–15 min chat or referral advice
- Close: gratitude

# Step 5 — Applications pipeline

## Funnel math (rule of thumb)
Applications → callbacks → interviews → offers  
Improve conversion at the **weakest** stage first.

## Tracking spreadsheet fields
- Company, role, location
- Date applied
- Source (job board/referral)
- Status
- Next action + date
- Notes (keywords, recruiter name)

---

# Step 6 — Interview preparation plan

## 2-week sprint plan (general)
**Daily (60–120 min):**
- 1 DSA question (focused category)
- 1 system/back-end topic drill
- 1 behavioral story practice

## What to prepare in advance
- 6 STAR stories:
  - conflict
  - failure
  - leadership
  - ambiguity
  - high impact
  - prioritization
- 2-minute project demo script

---

# Step 7 — Interviews: execution habits

## Before interview
- Read job post; map requirements → your proof
- Prepare 3 questions for interviewer
- Rehearse your pitch + project demo

## During technical interview
- Clarify requirements and constraints
- State approach + complexity
- Code cleanly; test with examples
- Communicate tradeoffs

## During behavioral
- Use STAR/SAO
- Quantify results
- End with learning and prevention

## After interview
- Send thank-you note (same day)
- Write a quick retro:
  - what went well
  - what to improve
  - what to study next

---

# Step 8 — Offer & negotiation basics (non-legal)

## Levers
- base salary
- equity/bonus
- start date
- scope/level/title
- remote/hybrid
- learning budget

## Negotiation stance
- be calm, grateful, and data-driven
- ask questions first, then counter with rationale
- never bluff; ask for time to decide
